ReAct:Synergizing Reasoning and Acting in Language Models

发表于 2022-10-17 分类于 论文精读 阅读次数：

Motivation

“chain-of-thought” reasoning:

uses its own internal representations to generate thoughts and is not grounded in the external world. This can lead to issues like fact hallucination and error propagation over the reasoning process.

planning and acting in interactive environments:

do not employ language models to reason abstractly about high-level goals or maintain a working memory to support acting.

There have not been studies on how reasoning and acting can be combined in a synergistic manner for general task solving.

Methodology

Action：refer to as a thought or a reasoning trace, does not affect the external environment.

Thought: compose useful information by reasoning over the current context $c_t$, and update the context $c_{t+1}$ to support future reasoning or acting.

PaLM-540B: is prompted with few-shot in-context examples to generate both domain-specific actions and free-form language thoughts.

Knowledge-Intensive Reasoning

Datasets: HotPotQA, FEVER

Actions: search[entity], lookup[string], finish[answer]

only retrieve a small part. Force models to retrieve via explicit reasoning in language.

ReAct Prompting: manually compose ReAct-format trajectories.

  1. use a combination of thoughts that decompose questions;
  2. extract information from Wikipedia observations;
  3. perform commonsense or arithmetic reasoning;
  4. guide search reformulation and synthesize the final answer

Baselines: Standard prompting, Chain-of-thought prompting (CoT) / self-consistency (CoT-SC), Acting-only prompt (Act)

Method: incorporate ReAct and CoT-SC, let the model decide when to switch to the other method.

Finetuning: consider a bootstraping approach

Conclusion:

  1. Hallucination is a serious problem for CoT, ReAct is more grounded, fact-driven, and trustworthy.
  2. ReAct repetitively generates the previous thoughts and actions, and fails to reason about what the proper next action to take and jump out of the loop.

Decision-Making

Datasets: ALFWorld, WebShop

ReAct Prompting: randomly annotate three trajectories from the training set for each task type.

  1. decompose the goal
  2. track subgoal completion
  3. determine the next subgoal
  4. reason via commonsense

Baselines: BUTLER

Conclusion:

ReAct combined reasoning and action using an LLM applied to an interactive environment within a closed-loop system.

  1. the thoughts is free-form and unlimited, through which human annotation can induce specific policies for the LLM to follow;
  2. the thoughts is abstract and diverse, including memory traces, strategization and error recovery;
  3. the thoughts is sparse, ReAct can decide when to think and avoid cluttering the LLM input history.

Allow a human to inspect and edit ReAct’s reasoning traces.

Reference
